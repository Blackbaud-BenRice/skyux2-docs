The split view component allows users to work through a list of items, while displaying each item's details in context. The split view component consists of two panels: the list panel, and the workspace panel. The SkySplitViewService allows consumers to control what is displayed in the workspace panel.

List type

Repeaters will be the most common component used for the list of items due to the flexible content layouts they enable.

Tree views can also be used if the items in the list are hierarchical. You may use different views in the workspace panel if the items at different levels of the tree are of different types.

While grids may technically be used, avoid them if possible. They do not scale well to the typically-small width of the navigation list.

List manipulation

You may include a list toolbar so that users can filter, sort and search the list, or add new items to the list.

An event that the split view component emits when it is about to hide the workspace panel. It emits a SkySplitViewBeforeWorkspaceCloseHandler object with a closeWorkspace method that closes the workspace panel. If a subscription exists for this event, the workspace does not hide until the subscriber calls the closeWorkspace method. This is typically used while in responsive mode to prevent a user from clicking the "Back to list" button and losing unsaved changes in their workspace.
