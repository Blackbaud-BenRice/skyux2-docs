The split view component allows users to work through a list of items, while displaying each item's details in context. The split view component consists of two panels: the list panel, and the workspace panel. The SkySplitViewService allows consumers to control what is displayed in the workspace panel.

Specifies the default width of the left-side list panel. This property accepts number values. Provides an observable to send commands to the split view component. The commands should respect the SkySplitViewMessage type. Sets SkySplitViewMessage to one of the following types:
  • SkySplitViewMessageType.FocusWorkspace — Sets focus on the first focussable element in the workspace.
  • SkySplitViewMessageType.EnableIteratorNextButton — Enables the next iterator button.
  • SkySplitViewMessageType.EnableIteratorPreviousButton — Enables the previous iterator button.
  • SkySplitViewMessageType.DisableIteratorNextButton — Disables the next iterator button.
  • SkySplitViewMessageType.DisableIteratorPreviousButton — Disables the previous iterator button.
An event that the split view component emits when it is about to hide the workspace panel. It emits a SkySplitViewBeforeWorkspaceCloseHandler object with a closeWorkspace method that closes the workspace panel. If a subscription exists for this event, the workspace does not hide until the subscriber calls the closeWorkspace method. This is typically used while in responsive mode to prevent a user from clicking the "Back to list" button and losing unsaved changes in their workspace. An event that the split view component emits when users click the next iterator button. An event that the split view component emits when users click the previous iterator button. Specifies content to display in the split view's list panel on the left. Specifies content to display in the split view's workspace panel on the right.
